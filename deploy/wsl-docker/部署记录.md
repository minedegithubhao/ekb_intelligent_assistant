# WSL Docker 服务部署记录

本文记录 KnowForge RAG 本地基础服务的部署方式，目标是让 MySQL、Redis、Milvus 在 WSL Docker 中可重复部署。生产部署时应使用同一份 `docker-compose.yml`、同一份环境变量文件、同一组镜像标签或镜像 digest。

## 1. 部署范围

| 服务 | 容器名 | 端口 | 数据卷 |
| --- | --- | --- | --- |
| MySQL | `knowforge-mysql` | `3306` | `knowforge-rag_mysql_data` |
| Redis | `knowforge-redis` | `6379` | `knowforge-rag_redis_data` |
| Milvus | `knowforge-milvus` | `19530`, `9091` | `knowforge-rag_milvus_data` |
| Etcd | `knowforge-etcd` | 仅 Docker 网络内 | `knowforge-rag_etcd_data` |
| MinIO | `knowforge-minio` | `9000`, `9001` | `knowforge-rag_minio_data` |

Milvus standalone 依赖 Etcd 和 MinIO，因此 Compose 中一并部署。

## 2. 当前镜像策略

本机访问 Docker Hub 认证接口出现连接重置，因此本地 `.env` 默认使用 Daocloud Docker Hub 镜像前缀；Etcd 使用 Quay。生产环境如果可以直接访问官方 registry，可以把 `.env` 中的镜像切换为官方引用。

当前使用的镜像变量：

```env
MYSQL_IMAGE=docker.m.daocloud.io/library/mysql:8.4
REDIS_IMAGE=docker.m.daocloud.io/library/redis:7.4
ETCD_IMAGE=quay.io/coreos/etcd:v3.5.18
MINIO_IMAGE=docker.m.daocloud.io/minio/minio:RELEASE.2024-12-18T13-15-44Z
MILVUS_IMAGE=docker.m.daocloud.io/milvusdb/milvus:v2.5.18
```

官方镜像等价写法：

```env
MYSQL_IMAGE=mysql:8.4
REDIS_IMAGE=redis:7.4
ETCD_IMAGE=quay.io/coreos/etcd:v3.5.18
MINIO_IMAGE=minio/minio:RELEASE.2024-12-18T13-15-44Z
MILVUS_IMAGE=milvusdb/milvus:v2.5.18
```

## 3. 首次部署步骤

从 Windows PowerShell 执行：

```powershell
cd E:\Heima-AI\ekb_intelligent_assistant
wsl -d Ubuntu -- bash -lc 'cd /mnt/e/Heima-AI/ekb_intelligent_assistant/deploy/wsl-docker && docker compose --env-file .env pull'
wsl -d Ubuntu -- bash -lc 'cd /mnt/e/Heima-AI/ekb_intelligent_assistant/deploy/wsl-docker && docker compose --env-file .env up -d'
```

查看容器状态：

```powershell
wsl -d Ubuntu -- bash -lc 'cd /mnt/e/Heima-AI/ekb_intelligent_assistant/deploy/wsl-docker && docker compose --env-file .env ps'
```

查看日志：

```powershell
wsl -d Ubuntu -- bash -lc 'cd /mnt/e/Heima-AI/ekb_intelligent_assistant/deploy/wsl-docker && docker compose --env-file .env logs -f mysql redis milvus'
```

## 4. 连通性验证

从 Windows PowerShell 使用当前 conda 环境验证：

```powershell
conda run -n knowforge-rag python -c "import pymysql; conn=pymysql.connect(host='127.0.0.1',port=3306,user='knowforge',password='knowforge_mysql_2026',database='knowforge_rag',charset='utf8mb4'); cur=conn.cursor(); cur.execute('select version()'); print(cur.fetchone()[0]); conn.close()"
conda run -n knowforge-rag python -c "import redis; r=redis.Redis(host='127.0.0.1',port=6379,password='knowforge_redis_2026',decode_responses=True); print(r.ping())"
conda run -n knowforge-rag python -c "from pymilvus import connections, utility; connections.connect(alias='default', host='127.0.0.1', port='19530'); print(utility.get_server_version()); connections.disconnect('default')"
```

## 5. 常用运维命令

停止服务但保留数据：

```powershell
wsl -d Ubuntu -- bash -lc 'cd /mnt/e/Heima-AI/ekb_intelligent_assistant/deploy/wsl-docker && docker compose --env-file .env down'
```

重启服务：

```powershell
wsl -d Ubuntu -- bash -lc 'cd /mnt/e/Heima-AI/ekb_intelligent_assistant/deploy/wsl-docker && docker compose --env-file .env up -d'
```

彻底删除服务和数据卷：

```powershell
wsl -d Ubuntu -- bash -lc 'cd /mnt/e/Heima-AI/ekb_intelligent_assistant/deploy/wsl-docker && docker compose --env-file .env down -v'
```

注意：`down -v` 会删除 MySQL、Redis、Milvus、Etcd、MinIO 数据，只能在确认不需要数据时执行。

## 6. 生产复现注意事项

- 复制 `.env.example` 为 `.env`，并替换所有密码。
- 保持 `docker-compose.yml` 和 `.env` 中的镜像版本一致。
- 生产环境建议将镜像引用替换为内部制品库地址或 digest 锁定地址。
- 数据卷默认由 Docker 管理，位于 WSL/Docker 的 Linux 存储中，不直接写入当前 `E:` 工作区。
- Redis 已启用 AOF，MySQL 使用 `utf8mb4`，Milvus 使用 standalone 模式。
