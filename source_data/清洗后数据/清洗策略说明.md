# 京东规则文档清洗策略说明

## 1. 清洗目标

本阶段只做原始 Markdown 文档清洗，不做文本切分、不做向量化、不做知识库入库。

目标是将采集到的京东规则文档整理为结构统一、可读性较好、便于后续 RAG 处理的干净数据，并保留原始文档粒度：一篇原始文档对应一篇清洗后文档。

## 4. 具体清洗规则

### 4.1 保留内容

每篇文档保留以下信息：

```text
标题
rule_id
source_url
label_names
active_time
update_time
正文内容
```

这些字段用于后续追溯来源、展示引用、过滤生效时间和更新时间。

### 4.2 删除噪声

清理以下无效或低价值内容：

```text
孤立的 |
目录锚点链接
空的 [] 残留
多余空行
BOM 等不可见字符
```

示例：

```text
|
```

会被删除。

示例：

```text
[第一章 概述](https://rule.jd.com/xxx#第一章概述)
```

会被删除或还原为普通文本。

### 4.3 链接处理

普通外部链接保留为可读文本：

```text
链接文本（URL）
```

本地锚点链接只保留链接文本：

```text
[附件1](#附件1)
```

清洗为：

```text
附件1
```

### 4.4 图片处理

图片不下载，按图片类型做文本化处理。

外部图片 URL 保留为普通文本。

示例：

```text
![image](https://img30.360buyimg.com/xxx.png)
```

清洗为：

```text
图片：https://img30.360buyimg.com/xxx.png
```

内嵌 base64 图片不保留原始编码，统一替换为占位文本。

示例：

```text
![image](data:image/png;base64,iVBORw0KGgo...)
```

清洗为：

```text
图片：内嵌图片已省略
```

这样可以避免超长 base64 字符串污染正文和后续检索，同时保留“此处原本有图片”的语义提示。

### 4.5 标题格式处理

对部分标题格式进行统一：

```text
**第一章 概述**
```

清洗为：

```text
## 第一章 概述
```

混合格式：

```text
# **第一章 概述**
```

清洗为：

```text
# 第一章 概述
```

### 4.6 表格处理

本批文档中存在大量网页表格。表格承载了指标项、适用范围、检验方法、类目规则等结构化信息，如果直接压成普通文本，会丢失行列关系。

当前策略改为：优先使用采集 JSON 中的原始 HTML 表格重新生成干净 HTML 表格，再进入清洗流程。

原因是 Markdown 表格不支持 `rowspan`、`colspan` 等合并单元格。对于《京东开放平台类目资费规则》这类大表，如果强行转成 Markdown 表格，会出现“列1/列2”和跨行提示混杂的问题，表格语义反而更乱。

清洗后表格示例：

```html
<table>
  <tr>
    <td rowspan="2">一级类目</td>
    <td rowspan="2">二级类目</td>
    <td colspan="4">保证金（SOP）</td>
  </tr>
  <tr>
    <td>GMV ＜5万</td>
    <td>5 万≤GMV＜10万</td>
    <td>10 万≤GMV＜30万</td>
    <td>GMV ≥30万</td>
  </tr>
</table>
```

清洗后的 HTML 表格会去掉 Word 样式、class、颜色等展示噪声，只保留 `table`、`tr`、`td`、`th` 结构，以及 `rowspan`、`colspan` 合并单元格属性。这样可以保留原始行列关系，后续人工查看和 RAG 解析都更稳定。

对于历史转换过程中产生的孤立竖线，仍作为格式噪声删除。例如：

```text
|
商业空间类目经营类型
|
涵盖经营对象
|
```

这类孤立竖线会被删除。

清洗后示例：

```text
商业空间类目经营类型
涵盖经营对象
商业空间
清包
仅服务
```

对于 HTML 中的跨行、跨列单元格，清洗后直接保留 `rowspan`、`colspan` 属性，不再追加“跨 N 行”文本提示。

## 5. 未处理事项

本阶段刻意不做以下处理：

```text
不做 chunk 切分
不做摘要生成
不做向量化
不做去重合并
不删除疑似低相关文档
不做图片 OCR
不保留 base64 图片编码
```

这些步骤留给后续知识库构建阶段处理。
