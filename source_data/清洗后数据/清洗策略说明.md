# 京东规则文档清洗策略说明

## 1. 清洗目标

本阶段只做原始 Markdown 文档清洗，不做文本切分、不做向量化、不做知识库入库。

目标是将采集到的京东规则文档整理为结构统一、可读性较好、便于后续 RAG 处理的干净数据，并保留原始文档粒度：一篇原始文档对应一篇清洗后文档。

## 4. 具体清洗规则

### 4.1 保留内容

每篇文档保留以下信息：

```text
标题
rule_id
source_url
label_names
active_time
update_time
正文内容
```

这些字段用于后续追溯来源、展示引用、过滤生效时间和更新时间。

### 4.2 删除噪声

清理以下无效或低价值内容：

```text
孤立的 |
目录锚点链接
空的 [] 残留
多余空行
BOM 等不可见字符
```

示例：

```text
|
```

会被删除。

示例：

```text
[第一章 概述](https://rule.jd.com/xxx#第一章概述)
```

会被删除或还原为普通文本。

### 4.3 链接处理

普通外部链接保留为可读文本：

```text
链接文本（URL）
```

本地锚点链接只保留链接文本：

```text
[附件1](#附件1)
```

清洗为：

```text
附件1
```

### 4.4 图片处理

图片不下载，只保留图片地址。

示例：

```text
![image](https://img30.360buyimg.com/xxx.png)
```

清洗为：

```text
图片：https://img30.360buyimg.com/xxx.png
```

这样可以避免图片 Markdown 干扰正文检索，同时保留后续 OCR 或人工查看的入口。

### 4.5 标题格式处理

对部分标题格式进行统一：

```text
**第一章 概述**
```

清洗为：

```text
## 第一章 概述
```

混合格式：

```text
# **第一章 概述**
```

清洗为：

```text
# 第一章 概述
```

### 4.6 表格处理

本批文档中存在大量从网页表格转换来的残留格式，例如：

```text
|
商业空间类目经营类型
|
涵盖经营对象
|
```

本次清洗策略不尝试恢复复杂 Markdown 表格，而是删除孤立竖线，保留表格中的文本内容。

清洗后示例：

```text
商业空间类目经营类型
涵盖经营对象
商业空间
清包
仅服务
```

该方式更适合后续 RAG 检索，因为语义文本被保留，格式噪声被减少。

## 5. 未处理事项

本阶段刻意不做以下处理：

```text
不做 chunk 切分
不做摘要生成
不做向量化
不做去重合并
不删除疑似低相关文档
不做图片 OCR
不恢复复杂表格结构
```

这些步骤留给后续知识库构建阶段处理。
