# 后端管理系统设计与接口文档

## 1. 项目背景

本项目当前已完成京东规则文档的采集、分类和清洗，后续需要建设一个后端系统，用于支撑用户登录、聊天会话、文档管理、知识库管理、版本管理、评估管理、参数管理，以及后续 RAG 问答能力接入。

本阶段重点是完成后端基础能力和接口约定，不实现前端页面，不展开完整 RAG 检索链路。聊天问答、评估和语音识别均预留服务抽象边界，方便后续替换真实模型服务。

## 2. 第一版系统目标

第一版后端系统目标如下：

- 提供用户登录接口，支持前端 MD5 加密密码后登录。
- 登录成功后返回 token，并在 Redis 中缓存用户信息、分类字段、可查询问题分类。
- 后端记录用户 session，用于 token 校验和登录状态管理。
- 支持用户聊天会话的新增、查询、删除。
- 支持获取指定会话的历史消息。
- 支持用户发送问题，后端保存用户消息和助手回答。
- 支持批量上传文档并保存完整原文。
- 支持版本管理，包括新建版本、版本切换、版本归档。
- 支持知识库管理，包括新建知识库、上传文档、删除文档。
- 支持评估管理，包括创建评估、执行评估、查看评估结果。
- 支持仪表台配置和参数管理查询。
- 支持实时语音识别。
- 保留 `RAGService` 抽象调用边界，后续可接入真实 RAG 能力。

## 6. 登录与缓存设计

### 6.1 登录流程

```http
POST /api/auth/login
```

请求参数：

```json
{
  "username": "employee001",
  "password": "前端MD5后的密码",
  "remember_me": true
}
```

后端处理逻辑：

1. 根据 `username` 查询用户。
2. 校验用户是否存在。
3. 校验用户是否启用。
4. 校验传入密码是否等于数据库中的 `password_hash`。
5. 根据用户部门查询可访问的问题分类。
6. 生成 JWT token。
7. 写入 MySQL `user_sessions`。
8. 将用户信息、分类字段、问题分类权限缓存到 Redis。
9. 返回 token。

响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "jwt-token",
    "token_type": "Bearer",
    "expires_in": 7200
  }
}
```

### 6.2 获取当前用户信息

```http
GET /api/auth/me
```

说明：

- 前端登录后可调用该接口获取用户展示信息。
- 后端优先从 Redis 读取用户缓存。
- Redis 未命中时回源 MySQL，并重新写入 Redis。

响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "username": "employee001",
    "name": "张三",
    "display_name": "张三",
    "department": "招商部",
    "category": "merchant",
    "is_admin": false,
    "is_active": true,
    "question_categories": ["企业店规则", "个人个体规则"]
  }
}
```

### 6.3 Redis 缓存设计

Redis Key 建议：

```text
auth:token:{jti} -> user_id
auth:user:{user_id} -> 用户信息 JSON
auth:user_categories:{user_id} -> 可查询问题分类 JSON
```

缓存内容：

```json
{
  "user_id": 1,
  "username": "employee001",
  "name": "张三",
  "department": "招商部",
  "category": "merchant",
  "is_admin": false,
  "question_categories": ["企业店规则", "个人个体规则"]
}
```

## 8. 用户端聊天接口

### 8.1 获取当前用户会话列表

```http
GET /api/conversations
```

查询参数：

```text
page
page_size
keyword
```

返回当前登录用户未逻辑删除的会话，按 `updated_at` 倒序排列。

### 8.2 新增聊天会话

```http
POST /api/conversations
```

请求参数：

```json
{
  "title": "新会话",
  "source_type": "rag"
}
```

`title` 不传时默认为 `新会话`。

### 8.3 获取会话历史消息

```http
GET /api/conversations/{conversation_id}/messages
```

说明：

- 只能查询当前登录用户自己的会话。
- 按消息创建时间升序返回。
- 只返回未逻辑删除消息。

### 8.4 删除聊天会话

```http
DELETE /api/conversations/{conversation_id}
```

说明：

- 逻辑删除会话。
- 同时逻辑删除该会话下所有历史消息。

### 8.5 聊天提问

```http
POST /api/conversations/{conversation_id}/messages
```

请求参数：

```json
{
  "message": "企业店入驻需要什么条件？",
  "question_category": "企业店规则",
  "use_rag": true,
  "knowledge_base_id": "kb_default",
  "stream": false
}
```

处理逻辑：

1. 校验 token、session 和 Redis 用户缓存。
2. 校验会话归属。
3. 校验问题分类权限。
4. 保存用户消息。
5. 如果会话标题为 `新会话`，使用问题前 20 个字更新标题。
6. 调用 `RAGService.answer(...)`。
7. 保存助手回答。
8. 返回回答和引用来源。

## 13. 实时语音识别接口

### 13.1 WebSocket 实时识别

```text
WS /api/asr/stream?token=<access_token>
```

说明：

- 前端建立 WebSocket 连接。
- 前端持续发送音频二进制片段。
- 后端调用抽象 `ASRService`。
- 后端实时返回中间识别结果和最终识别结果。

前端发送：

```text
binary audio chunk
```

后端返回中间结果：

```json
{
  "type": "partial",
  "text": "企业店入驻"
}
```

后端返回最终结果：

```json
{
  "type": "final",
  "text": "企业店入驻需要什么条件"
}
```

异常返回：

```json
{
  "type": "error",
  "message": "语音识别服务异常"
}
```

### 13.2 ASR 服务边界

建议抽象接口：

```python
class ASRService:
    async def accept_audio_chunk(self, session_id: str, chunk: bytes) -> dict:
        ...

    async def close_session(self, session_id: str) -> dict:
        ...
```

第一版只定义接口边界，真实语音识别引擎后续接入。



## 18. 测试与验收标准

### 18.1 登录

- 正确账号密码登录成功。
- 错误密码登录失败。
- 禁用用户登录失败。
- 登录成功后写入 MySQL session。
- 登录成功后写入 Redis 用户缓存。
- token 过期后接口返回 401。

### 18.2 会话与消息

- 能新增会话。
- 能查询当前用户历史会话。
- 不能查询其他用户会话。
- 删除会话后列表不再展示。
- 删除会话后消息逻辑删除。
- 提问后保存用户消息和助手消息。

### 18.3 知识库

- 能新建知识库。
- 能查询知识库列表。
- 能上传文档到指定知识库。
- 能逻辑删除知识库文档。
- 能归档知识库。

### 18.4 版本

- 能新建版本。
- 能查询版本列表。
- 能切换 active 版本。
- 能归档非 active 版本。

### 18.5 评估

- 能创建评估任务。
- 能执行评估任务。
- 能查看评估列表。
- 能查看评估详情。

### 18.6 仪表台与参数

- 能查询当前模型、重排、TopK 等配置。
- 能查询参数列表。
- 能更新参数。

### 18.7 实时语音识别

- WebSocket 能建立连接。
- 能接收音频片段。
- 能返回 partial 识别结果。
- 能返回 final 识别结果。
- 服务异常时返回 error 消息。
